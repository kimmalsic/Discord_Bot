# 사업관리 Discord 봇 기능 구체화

> 목적: **사업(프로젝트) 진행 상황을 Discord 채널에서 명확하게 관리**하고,  
> 기록·책임·의사결정을 자동화하여 Slack 수준 이상의 사업관리 환경을 구축한다.

---

## 1. 기본 전제

- 사용 환경: Discord 서버
- 채널 성격: **사업관리 전용 채널**
- 대상 사용자: PM, 팀장, 관리자, 실무자
- 명령어 방식: **한국어 Slash Command**
- 운영 봇 구성:
  - Carl-bot: 역할, 권한, 로그
  - 개인 개발 봇: 사업관리 기능 전담

---

## 2. 핵심 엔티티 정의

### 2.1 사업(Project)
- 사업은 관리의 최상위 단위
- 모든 일정·이슈·결정은 사업에 귀속됨

**필드**
- 사업 ID
- 사업명
- 사업 설명
- 담당 PM
- 참여 부서/인원
- 시작일
- 종료 예정일
- 상태
  - 기획
  - 진행중
  - 이슈
  - 보류
  - 완료
- 생성일 / 수정일

---

### 2.2 일정 / 마일스톤(Milestone)
- 사업 내 주요 일정 관리

**필드**
- 일정 ID
- 사업 ID
- 일정명
- 설명
- 목표 일자
- 담당자
- 상태 (예정 / 완료 / 지연)

---

### 2.3 이슈 / 리스크(Issue)
- 사업 진행 중 발생하는 문제 및 위험 요소

**필드**
- 이슈 ID
- 사업 ID
- 제목
- 상세 내용
- 영향도 (Low / Medium / High / Critical)
- 담당자
- 상태 (오픈 / 조치중 / 해결 / 종료)
- 발생일
- 해결일

---

### 2.4 의사결정(Decision)
- 공식 결정 사항 기록용

**필드**
- 결정 ID
- 사업 ID
- 결정 내용
- 결정 사유
- 결정자
- 결정 일시
- 관련 문서/링크

---

### 2.5 문서(Document)
- 사업 관련 자료 관리

**필드**
- 문서 ID
- 사업 ID
- 문서명
- 유형 (계약서 / 제안서 / 회의록 등)
- 링크 또는 파일
- 등록자
- 등록일

---

## 3. 기능 상세 (명령어 기준)

---

## 3.1 사업 관리 기능

### /사업 등록
- 신규 사업 생성

입력 항목:
- 사업명
- 담당 PM
- 시작일
- 종료 예정일
- 설명

결과:
- 사업 생성
- 요약 메시지 출력

---

### /사업 목록
- 전체 사업 리스트 조회

옵션:
- 상태별 필터
- 담당자별 필터

---

### /사업 상세
- 특정 사업 상세 정보 출력

출력 정보:
- 기본 정보
- 현재 상태
- 일정 요약
- 미해결 이슈 수

---

### /사업 상태변경
- 사업 상태 변경

상태:
- 기획 / 진행중 / 이슈 / 보류 / 완료

---

### /사업 종료
- 사업 종료 처리
- 완료 상태로 변경
- 종료 기록 저장

---

## 3.2 일정 / 마일스톤 기능

### /일정 추가
- 사업에 일정 추가

입력 항목:
- 사업 선택
- 일정명
- 목표 일자
- 담당자

---

### /일정 목록
- 사업별 일정 조회

옵션:
- 예정 일정만 보기
- 지연 일정 보기

---

### /일정 완료
- 일정 완료 처리
- 완료 로그 저장

---

## 3.3 이슈 / 리스크 기능

### /이슈 등록
- 신규 이슈 등록

입력 항목:
- 사업 선택
- 제목
- 상세 내용
- 영향도
- 담당자

특이사항:
- 영향도 Critical 시 관리자 자동 멘션

---

### /이슈 목록
- 사업별 이슈 조회

옵션:
- 미해결 이슈
- 영향도별 필터

---

### /이슈 조치
- 이슈 상태 변경

상태:
- 조치중 / 해결

---

### /이슈 종료
- 이슈 종료 처리
- 최종 조치 내용 기록

---

## 3.4 의사결정 관리

### /결정 기록
- 의사결정 사항 기록

입력 항목:
- 사업 선택
- 결정 내용
- 결정 사유
- 결정자

---

### /결정 목록
- 사업별 의사결정 목록 조회

---

## 3.5 문서 관리

### /문서 등록
- 문서 링크 또는 파일 등록

입력 항목:
- 사업 선택
- 문서명
- 유형
- 링크 또는 파일

---

### /문서 목록
- 사업별 문서 목록 조회

---

## 3.6 요약 & 리포트

### /요약
- 전체 사업 상태 요약

출력 예:
- 진행중 사업 수
- 이슈 상태 사업 수
- 지연 일정 수

---

### /주간요약 (자동)
- 주 1회 자동 실행

포함 내용:
- 상태 변경 사업
- 신규 이슈
- 해결 이슈
- 다가오는 일정

---

## 4. 알림 & 자동화 정책

- 일정 D-7 / D-1 알림
- 일정 지연 시 PM 멘션
- Critical 이슈 즉시 알림
- 3일 이상 미조치 이슈 경고

---

## 5. 권한 정책

- 관리자: 전체 권한
- PM: 사업/일정/이슈 관리
- 일반 사용자: 조회 및 등록만 가능

---

## 6. 단계별 개발 로드맵

### Phase 1 (필수)
- 사업 관리
- 일정 관리
- 이슈 관리

### Phase 2 (확장)
- 의사결정 기록
- 문서 관리
- 요약 리포트

### Phase 3 (고급)
- 통계
- AI 요약
- 대시보드 자동화

---

## 7. 최종 목표

> Discord를 단순 채팅 툴이 아닌  
> **"사업 상태가 자동으로 관리·기록되는 업무 시스템"**으로 사용한다.
